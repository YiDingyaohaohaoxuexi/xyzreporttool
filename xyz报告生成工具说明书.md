# xyz 报告生成工具说明书

## 1. 工具概述

本工具是一个单文件前端报表工具，主文件为：

- `xyz报告生成器v1内测版.html`

工具用途：

- 连接机器人/工控电脑提供的 API
- 按时间范围读取订单与任务数据
- 在页面中手工补充、修正部分报表字段
- 记录异常信息（文字、图片、视频）
- 填写 PM / FAE 联系信息
- 导出 Dashboard HTML、ZIP、Excel
- 重新读回已导出的 Dashboard，继续追加后续订单

本工具没有独立后端。  
打开页面的设备会直接访问工控电脑 API。

---

## 2. 使用环境

### 2.1 推荐环境

- Windows 电脑
- Chromium 内核浏览器
  - Google Chrome
  - Microsoft Edge

### 2.2 不同设备说明

- **电脑端**
  - 适合完整使用，包括查询、编辑、异常记录、导出
  - 推荐作为主工作端

- **手机端**
  - 更适合查看导出的 Dashboard
  - 不建议作为主要编辑端
  - 若手机无法连接 API，通常是网络访问或浏览器安全限制问题，不一定是工具本身故障

### 2.3 网络要求

使用本工具时，当前设备必须能够直接访问工控电脑 API。

也就是说：

- 电脑使用时：电脑浏览器 -> 工控电脑 API
- 手机使用时：手机浏览器 -> 工控电脑 API

不是通过别的电脑中转。

## 4. 基本使用流程

### 4.1 首次生成报表

1. 打开 `xyz报告生成器v1内测版.html`
2. 填写工控电脑的 IP 和端口
3. 点击“接续确认”
4. 选择作业模式
5. 选择时间范围
6. 设置 KPI / 标题 / 报表模式
7. 点击“获取数据”
8. 在订单编辑区补充手工字段
9. 需要时记录异常
10. 填写 PM / FAE 信息
11. 填写总结
12. 导出 HTML / ZIP / Excel

### 4.2 从旧 Dashboard 继续追加数据

1. 点击“Dashboard 读入”
2. 选择以前导出的 Dashboard HTML
3. 工具会恢复：
   - 已有订单
   - 手工编辑字段
   - 异常记录
   - 总结
   - PM / FAE 信息
4. 再次点击“获取数据”
5. 工具会读取新订单候选
6. 选择需要追加的订单
7. 再次导出

### 4.3 用多个日报生成周级数据

1. 先准备多个已导出的日报 Dashboard HTML
2. 点击“日报一括读入”
3. 选择多个日报文件
4. 工具会合并：
   - 订单数据
   - 异常记录
   - 每日报告总结
5. 合并后使用通常报表形式导出周级报表

注意：

- 批量读入的日报必须属于同一作业模式
- 不能混用卸货和码垛数据

## 5. 主要功能说明

### 5.1 作业模式

#### 荷降ろし（卸货）

适用于卸货/拆柜场景。

特点：

- 显示容器种别
- 显示卸货相关 KPI
- 支持时间达成率判断

#### パレタイズ（码垛）

适用于码垛/堆放场景。

特点：

- 字段结构与卸货模式不同
- 会显示托盘相关字段

### 5.2 Dashboard 形式

#### 通常レポート

适合多日或周级数据。

特点：

- 包含趋势图
- 包含完整明细与 KPI

#### 日報モード

适合单日数据。

特点：

- 省略趋势图
- 更偏向单日报表展示
- 页头只显示目标日期

### 5.3 干预时间限制

用于控制 PPH 计算时是否把“机器人介入时间”算入。

- OFF：
  - 使用旧逻辑
  - 仅显示普通运行时间

- ON：
  - 按设定的“每次介入分钟数”参与计算
  - 会影响 PPH 计算结果

### 5.4 异常记录

异常记录支持图文视频混合。

适用场景：

- 现场故障记录
- 原因追踪
- 处理结果留档

导出后会进入 Dashboard 中的异常记录区。

### 5.5 PM / FAE 信息

建议写法示例：

#### PM 信息

```text
张三
PM / Project Manager
ABC Team
zhangsan@example.com
090-1234-5678
```

#### FAE 信息

```text
李四
FAE / Field Application Engineer
XYZ Team
lisi@example.com
090-8888-9999
```

## 6. 导出说明

### 6.1 HTML 导出

生成一个可独立打开的 Dashboard HTML 文件。

适合：

- 浏览器查看
- 内部共享
- 作为后续继续追加数据的基础文件

### 6.2 HTML 保存/覆盖

用于把当前结果直接覆盖保存到已有 HTML 文件。

说明：

- 依赖浏览器文件保存能力
- 推荐使用 Chromium 内核浏览器
- 部分浏览器或手机浏览器可能不支持

### 6.3 ZIP 导出

用于导出带媒体资源的打包结果。

说明：

- 若异常记录里包含图片/视频，ZIP 更适合归档
- 可选择视频压缩
- 首次压缩时可能加载较慢

### 6.4 Excel 导出

导出为 `.xls` 格式。

内容包括：

- 标题
- 时间范围
- KPI 信息
- 订单明细
- PM / FAE 信息（如已填写）

---

## 7. Dashboard 回读说明

导出的 Dashboard HTML 内部会嵌入恢复用快照数据。

因此工具可以在后续重新读回 HTML 时恢复：

- 报表模式
- KPI 目标
- 订单行
- 手工编辑字段
- 异常记录
- 总结内容
- PM / FAE 信息

这也是推荐优先保留 HTML 导出文件的原因。

## 8. 常见问题

### 8.1 为什么电脑能连，手机不能连？

因为本工具是前端直连工控电脑 API。

常见原因：

- 手机虽然连了 Wi-Fi，但不能访问工控电脑内网
- 工控电脑防火墙拦截
- 服务只允许本机访问
- 手机浏览器限制访问内网 HTTP 接口

### 8.2 为什么“HTML 保存/覆盖”不能用？

常见原因：

- 当前浏览器不支持文件系统保存接口
- 手机浏览器不支持
- 使用了兼容性较弱的浏览器

建议：

- 改用 Chrome 或 Edge
- 或直接使用“HTML 导出”

### 8.3 为什么读回 Dashboard 后还要再查一次数据？

因为“读回 Dashboard”恢复的是旧快照。  
如果你想继续追加后续订单，需要再次点击“获取数据”去查询新数据。

### 8.4 为什么手机上体验不如电脑？

因为本工具包含：

- 表格编辑
- 多区域输入
- 明细滚动
- 媒体处理

这些都更适合在电脑上进行。

## 9. 推荐操作习惯

建议团队按以下方式使用：

1. 日常编辑与导出尽量在电脑端完成
2. 每次生成完成后保留 HTML 导出文件
3. 继续做下一版报表时优先读回旧 Dashboard
4. PM / FAE 信息按统一格式填写，方便对外共享
5. 若包含较多异常视频，优先使用 ZIP 导出归档

## 10. 维护说明

当前仓库的核心是单文件页面：

- `xyz报告生成器v1内测版.html`

如果后续继续扩展功能，优先检查以下内容是否同步更新：

- 主页输入区
- Excel 导出
- Dashboard HTML 导出
- Dashboard snapshot 导入/导出

- 页面能填，但导出没有
- 导出有，但读回丢失
- 主页面和 Dashboard 页面显示不一致
